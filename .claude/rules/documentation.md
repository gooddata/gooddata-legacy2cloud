# GoodData Migration Toolkit - Documentation Guidelines

## Docstring Requirements

- Every new function/method: 1-2 line concise docstring
- Every new class: brief description of purpose and responsibility
- Every new module: high-level overview at the top
- Public APIs (entry points, Builder methods): Google-style with Args/Returns/Raises

## README.md Requirements

All new features must be documented in README.md:
- New migration flows or workflows
- New CLI arguments (all of them)
- New output files and what they contain
- Follow the existing README.md structure and update Table of Contents if needed
